## 重要：今回の分析スコープ

今回の対象は、**SwitchBot Cloud API を呼び出す低レイヤーAPIクライアント**です。

このクラスの責務は、以下に限定します。

- SwitchBot API v1.1 の署名付きHTTPリクエストを送る
- デバイス一覧を取得する
- シーン一覧を取得する
- 指定されたシーンIDのシーンを実行する
- HTTPステータス、レスポンス本文、通信失敗を呼び出し元へ正しく返す

将来的には別システムからこのクラスが呼ばれる可能性がありますが、  
**今回の `SwitchBotApisV2.java` には、呼び出し元システム固有の責務を入れないでください。**

### 4. `SwitchBotApisV2.java` の設計方針

後続工程で `SwitchBotApisV2.java` を作る場合の方針を提案してください。

ただし、まだ実装しないでください。

方針には以下を含めてください。

- 認証情報の扱い
  - ソースコードべた書きを廃止する
  - コンストラクタ注入を基本にするべきか
  - 環境変数やpropertiesは、このクラスが直接読むべきか、それとも外側で読んで注入すべきか

  - フィールド保持か
  - コンストラクタ注入か
  - タイムアウト設定をどうするか
- GET / POST の共通化
  - 署名生成
  - ヘッダー生成
  - HTTPメソッド指定
  - Body有無
- 例外設計
  - 空文字返却をやめる
  - 専用例外を作るべきか
  - 通信失敗、認証失敗、HTTPエラーをどう呼び出し元へ伝えるか
- 戻り値設計
  - raw JSON string をそのまま返すべきか
  - HTTPステータスとbodyを持つ結果オブジェクトを返すべきか
  - V2として適切な外部仕様は何か
- APIパス管理
  - 内部クラスでよいか
  - 定数でよいか
  - enum化する必要があるか
