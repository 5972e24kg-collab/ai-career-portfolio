# Work068: 自律歩行システムとR3F空間補正

Tag: [[React Three Fiber]], [[VRM]], [[Animation]], [[Autonomous Agent]]

## 1. 目的

* ジェムちゃんをR3Fの3D空間内で自律的に移動させ、人間の「間」を持った自然な歩行・停止・振り向き動作を実装する。

## 2. システム環境

* **Node:** Client（開発端末）
* **Target:** `VrmGemchanAvatar.js`, `useAvatarMotionControl.js` (React Three Fiber / @pixiv/three-vrm)

## 3. 作業ログ・解決プロセス

### 発生した課題

* アニメーションのRoot Motion（前進）とプログラムのLerp移動が競合し、キャラクターが吹き飛ぶ。
* Lerpによる移動計算を用いると、歩き出しが音速になり徐々に減速する「足滑り（瞬間移動）」が発生する。
* 到着時の「振り向き」と「移動」が同時に行われるため横滑りする。
* 画面端へ移動した際、FOV45の透視投影カメラの影響で「明後日の方向」を向いているように描画される。
* 初回のモーションファイルロード時に数秒のラグが生じ、非同期制御がすれ違ってキャンセルされる。

### 解決策

* HipsボーンのX/Z軸移動を完全に無効化し、Y軸のバウンスだけを残す「真・In-Placeパッチ」の実装。
* キャラクターのX軸移動をLerpから等速直線運動（`Math.sign` + `step`）へ変更。
* X座標に応じた内側への視線補正角（`vrm.scene.position.x * 0.4`）を加算し、常にユーザーの目を見つめ返す処理を追加。
* `useAvatarMotionControl` 側に、物理側が `isMoving = true` を返すまで待機（最大10秒のタイムアウト付き）するポーリング制御を実装。

## 4. 成果物

* `play_Move(targetLogicalX)` コマンドを呼ぶだけで、バストアップからの全身切り替え・カメラ引き・指定座標への歩行・待機状態への復帰までの一連の動作を自律的に実行するシステム基盤。
