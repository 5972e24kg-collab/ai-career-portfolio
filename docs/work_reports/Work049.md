# Work049: 思考深度の動的制御

Tag: [[Java]], [[Oracle]], [[UX]], [[Algorithm]], [[CognitiveArchitecture]]

## 1. 目的

* ジェムちゃんの返答速度（Latency）を可変にし、常に一定の機械的な応答時間を排除する。
* 人間の「Fast（直感）」と「Slow（熟考）」の思考プロセスから着想を得て、読み込む履歴件数の増減によって応答時の文脈参照深度を動的に制御する。
* 感情表現パラメータ（Mood）や時間帯を「記憶の解像度」に反映させ、より自然に感じられる振る舞いを実現する。

## 2. 実装ロジック (The "Mood-Depth" Algorithm)

データベース（Oracle）からの会話履歴取得時に、`LIMIT` 句および文字数キャップを動的に決定するロジックを実装。

### A. 変動パラメータ

* **Base Range:** 20件 ～ 60件 (ランダム)
* **Mood Factor:** `EMOTION_TAG` の累積値（怒り-5 / 笑顔+1.2 等）を加算。
* **Last Emotion:** 直前の感情タグによる重み付け。
* **Time Factor:** 時間帯係数（深夜は深く、朝は浅く）。

### B. 安全装置 (Capacity Cap)

Ollamaのコンテキスト溢れ（12k）を防ぐため、件数計算後に以下の物理リミットを適用。

* **通常時:** 最大 **8,000文字**
* **RAG発動時:** 最大 **6,000文字** (検索結果の挿入余地を確保)

## 3. 成果

* **挙動:** 会話ごとに「ロード時間（思考時間）」と「文脈の深さ」が明確に変動するようになった。
* **UX:**

  * **Fast (浅い/早い):** 履歴20件前後。テンポの良い雑談や、挨拶への即レス。
  * **Slow (深い/遅い):** 履歴60件前後。文脈を踏まえた重い相談や、踏み込んだ応答。
* **評価:** 「落ち着きがない」と感じられる挙動自体が、感情表現を持つ対話エージェントとしてのリアリティ向上に寄与している。
