# Work045: Invisible Directorと一時的な状態変化の付与

Tag: [[Java]], [[SystemPrompt]], [[MultiAgent]], [[ChaosEngineering]]

## 1. 目的

* ジェムちゃん (27B) に対し、継続的な性格変更ではなく、**「その瞬間だけ湧き上がる衝動（魔が差す瞬間）」**を実装する。
* 指示そのものをコンテキスト（短期記憶）に残さないことで、発言後に「なぜそんなことを言ったのか自分でも分からない」ように見える、人間的な不条理さを再現する。
* 開発者ですら予測しきれない「3者間の認識齟齬」によるエンターテインメント性を観測する。

## 2. システム環境

* **Node:** ローカル検証ノード（大容量VRAM環境）
* **Director:** `gpt-oss:120b` (Stateless Instruction Generator)
* **Actor:** `GemChan5.1:27b` (Stateful Context Aware)
* **Mechanism:** Javaによる `[DIRECTOR'S ORDER]` の動的注入と即時破棄。

## 3. 作業ログ・解決プロセス

### 実装した仕様 (The 23-Hour Ghost)

1. **Invisible Injection:**

   * Java側でユーザーメッセージ直前に `[DIRECTOR'S ORDER]` を結合して推論させるが、**Oracle DBに保存する会話履歴からはこの指示ブロックを削除する**。
   * これにより、次回推論時のコンテキストには、その指示が残らない。
2. **Cycle Shifting:**

   * 発動周期を「23時間」に設定。毎日1時間ずつズレることで、ユーザーの生活リズムに対して「神出鬼没」なタイミングで介入する。

### 挙動検証 (The "Work BGM" Incident)

* **Order:** `MISSION_TOPIC: 作業BGM`, `FORCE_EMOTION: Dere 70%`
* **Result:**

  * ジェムちゃんが**「貴方の声（ノイズ）を作業用BGMにする」**という、高度かつ意味深なデレ発言を生成。
  * 指示（BGM）と感情（デレ）を見事に統合したが、その動機はログに残らず、**「突然デレて意味深な供述をした彼女」**という挙動だけが観測された。

## 4. 成果

* **感情揺らぎの演出:** 「理由のない機嫌の良さ/悪さ」に見える挙動を、システム的に再現できることを確認。
* **混乱のエンタメ化:** Director（指示を保存しない）、Actor（次回コンテキストでは動機を参照できない）、User（全体構造を知る）という非対称な情報構造を確認できた。

## 5. 今後の展望

* **カオス・モニタリング:** gpt-oss君が、自分が過去に出した指示によって歪んだ会話ログを見て、次にどのような「勘違いした演出」を出してくるかを観察する。
